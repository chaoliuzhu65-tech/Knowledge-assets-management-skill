# 飞书知识资产沉淀 Skill

基于飞书 MCP 插件能力，实现经验快速沉淀、知识结构化、自动归档和复用的全流程管理。

## 🎯 核心价值
- **效率提升**：将群聊讨论、会议记录一键转化为结构化知识资产，节省80%的整理时间
- **知识复用**：建立组织级知识库，避免重复踩坑，新项目可快速参考已有经验
- **资产沉淀**：将零散的沟通内容转化为可复用的组织资产，实现知识的持续积累
- **自动归档**：自动归档到指定知识库，建立统一的知识管理体系

## 📦 目录结构
```
feishu-knowledge-assets/
├── SKILL.md                          # 主技能文档，核心工作流和使用说明
├── scripts/

├── references/
│   ├── archive_config.md             # 归档路径配置，知识库和多维表信息
│   ├── domain_classification.md      # 领域分类和标签规范
│   └── templates/

│       ├── incident_review.md        # 故障复盘模板
│       └── best_practice.md          # 最佳实践模板
└── README.md                         # 本说明文档
```

## 🚀 快速开始
### 1. 基础配置
1. 参考 `references/archive_config.md` 配置知识库和多维表信息
2. 根据组织需求调整 `references/domain_classification.md` 中的分类和标签
3. 定制 `references/templates/` 下的模板，适配组织需求

### 2. 使用示例
#### 示例1：生成项目经验总结
```
用户：帮我总结下项目群最近一周的讨论，生成经验总结
```
- 自动获取群聊历史消息
- 提取核心观点、决策、经验教训
- 按项目总结模板结构化生成文档
- 自动归档到知识库并写入索引表
- 返回文档链接

#### 示例2：生成故障复盘报告
```
用户：根据这个故障群的聊天记录，生成故障复盘报告
```
- 自动梳理故障时间线
- 提取根因分析和处理过程
- 自动生成改进措施列表
- 关联相关资源（监控截图、会议录音等）
- 归档到技术知识库

#### 示例3：知识复用
```
用户：有没有类似支付系统故障的处理经验？
```
- 搜索知识资产索引表
- 全文搜索相关文档
- 返回最相关的3份经验文档
- 支持直接整合生成新的解决方案

## 🛠️ 工具依赖
- feishu_im_user_get_messages：读取群聊历史消息
- feishu_im_user_get_thread_messages：展开话题回复
- feishu_create_doc：创建知识文档
- feishu_update_doc：更新文档内容
- feishu_bitable_app_table_record：维护知识资产索引

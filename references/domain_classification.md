# 知识资产领域分类规范

- 架构设计
- 开发规范
- 故障处理
- 性能优化
- 安全合规
- 运维实践
- 数据技术
- AI/大模型应用

### 产品类（Product）
- 需求分析
- 用户研究
- 产品设计
- 迭代规划
- 数据分析
- 商业化
- 用户运营

### 项目类（Project）
- 项目管理
- 敏捷实践
- 需求管理
- 风险管理
- 质量保障
- 跨团队协作
- 项目复盘

### 运营类（Operations）

- 用户增长
- 内容运营
- 社区运营
- 品牌营销

### 管理类（Management）

- 人才招聘
- 绩效管理
- 企业文化
- 流程优化
- 战略规划
- 财务管理

## 标签规范
### 关键词标签要求
- 最多5个标签
- 优先使用已有标签，避免重复
- 标签颗粒度适中，避免过于宽泛或具体
- 中英文统一，优先使用中文

### 常用标签示例
- 技术类：Python, Go, MySQL, Redis, Kubernetes, 微服务, 分布式, 性能优化, 故障排查
- 产品类：用户体验, 需求分析, A/B测试, 增长黑客, 用户画像, 产品 roadmap
- 项目类：敏捷开发, Scrum, 需求管理, 风险管理, 跨团队协作, 项目复盘
- 运营类：活动策划, 用户增长, 内容营销, 社群运营, 转化率, 留存率
- 管理类：OKR, 绩效考核, 人才培养, 组织架构, 流程优化, 企业文化

## 命名规范
### 文档标题格式
```
[领域] [类型]：标题 - [YYYY-MM-DD]
```

### 示例
- `[技术] [故障复盘]：支付系统超时问题处理 - 2026-03-14`
- `[产品] [最佳实践]：用户注册流程优化 - 2026-03-10`
- `[项目] [经验总结]：XX项目一期上线复盘 - 2026-03-01`
- `[运营] [活动方案]：618大促活动策划 - 2026-02-20`
